## 📊 3. PHASE 2: Data Understanding (4 ມື້)

### 3.1 Data Model (ERD)
- 6 Tables ຫຼັກ
- Primary Keys: UUID
- Foreign Keys: ເຊື່ອມໂຍງທຸກ table

### 3.2 Data Dictionary

#### Table: employees
| Field | Type | Description | Validation |
|-------|------|-------------|------------|
| id | UUID | Primary Key | Auto-generate |
| emp_code | String | ລະຫັດພະນັກງານ | Unique, EMP-XXXX |
| full_name | String | ຊື່-ນາມສະກຸນ | Required, 2-100 chars |
| email | String | ອີເມລ | Valid email format |
| phone | String | ເບີໂທ | 020XXXXXXXX |
| department_id | UUID | FK to departments | Required |
| position | String | ຕຳແໜ່ງ | Required |
| base_salary | Decimal | ເງິນເດືອນພື້ນຖານ | > 0 |
| role | Enum | user/admin | Default: user |

#### Table: attendance
| Field | Type | Description | Validation |
|-------|------|-------------|------------|
| id | UUID | Primary Key | Auto-generate |
| employee_id | UUID | FK to employees | Required |
| record_date | Date | ວັນທີບັນທຶກ | YYYY-MM-DD |
| check_in | Time | ເວລາເຂົ້າ | HH:MM:SS |
| check_out | Time | ເວລາອອກ | HH:MM:SS, > check_in |
| check_in_lat | Decimal | Latitude | -90 to 90 |
| check_in_lng | Decimal | Longitude | -180 to 180 |
| status | Enum | on_time/late/absent | Auto-calculate |
| work_hours | Decimal | ຊົ່ວໂມງ | Auto-calculate |

### 3.3 Data Flow

```
[Mobile App] → [GPS + Selfie] → [Validation] → [Google Sheets]
                                      ↓
[Admin Panel] ← [Dashboard] ← [Aggregation] ← [Raw Data]
                                      ↓
                              [Export Excel]
```

## 🔧 4. PHASE 3: Data Preparation (5 ມື້)

### 4.1 Google Sheets Structure

```

├── 📄 employees (Sheet 1)
├── 📄 attendance (Sheet 2)
├── 📄 leave_requests (Sheet 3)

├── 📄 payroll (Sheet 5)
├── 📄 departments (Sheet 6)
└── 📄 config (Sheet 7 - ຕັ້ງຄ່າລະບົບ)
```

### 4.2 Data Validation Rules

| Field | Rule | Error Message |
|-------|------|---------------|
| Email | Valid format | "ອີເມລບໍ່ຖືກຕ້ອງ" |
| Phone | 020/030 + 8 digits | "ເບີໂທບໍ່ຖືກຕ້ອງ" |
| Date | YYYY-MM-DD | "ຮູບແບບວັນທີບໍ່ຖືກ" |
| Salary | Number > 0 | "ເງິນເດືອນຕ້ອງ > 0" |
| GPS | Within 100m | "ຢູ່ນອກຂອບເຂດຫ້ອງການ" |

### 4.3 Auto-Calculation Formulas

```javascript
// ຄຳນວນຊົ່ວໂມງເຮັດວຽກ
work_hours = (check_out - check_in) - lunch_break

// ຄຳນວນສະຖານະ
status = check_in <= "08:15" ? "on_time" : "late"

// ຄຳນວນເງິນ OT
ot_amount = ot_hours * (base_salary / 176) * 1.5

// ຄຳນວນເງິນເດືອນສຸດທິ
net_salary = base_salary + allowance + ot_amount + bonus 
           - tax - social_security - other_deductions
```

## 🚀 7. PHASE 6: Deployment (3 ມື້)

### 7.1 Deployment Checklist

- [ ] Deploy PWA to Vercel
- [ ] Deploy Apps Script to Google Cloud
- [ ] Configure custom domain (optional)
- [ ] Setup SSL certificate (auto by Vercel)
- [ ] Configure CORS
- [ ] Setup error logging
- [ ] Create user accounts
- [ ] Import initial data
- [ ] User training (2-3 hours)
- [ ] Go-live!

### 7.2 Monitoring Setup

| Metric | Tool | Alert Threshold |
|--------|------|-----------------|
| Uptime | UptimeRobot (Free) | < 99% |
| Errors | Apps Script Logs | > 5/hour |
| Response Time | Custom logging | > 3 seconds |
| Active Users | Google Analytics | < 50%/day |

### 7.3 Rollback Plan

1. Keep backup of Google Sheets (auto by Google)
2. Version control Apps Script (Git)
3. Vercel auto-rollback on failure
4. Document manual rollback steps
